# Analytics

## Overview

The Analytics module provides comprehensive insights into the performance of your AI assistants by analyzing conversations and user interactions. This feature helps you understand how well your assistants are performing, identify areas for improvement, and track key metrics over time.

## Notions

`Score` - measures the overall quality and effectiveness of the conversation on a scale of 1-10.

`Performance` - indicates the overall effectiveness of the agent's responses and actions during the conversation, categorized as:
- good: Agent consistently provided accurate and helpful responses
- average: Agent performed adequately but with room for improvement
- poor: Agent struggled to provide appropriate or helpful responses
- unknown: Not enough data to evaluate performance

`Sentiment` - measures the emotional tone of the conversation between user and agent:
- positive: User shows satisfaction and positive emotions
- neutral: Conversation maintains a balanced, professional tone
- negative: User expresses dissatisfaction or frustration
- unknown: Sentiment cannot be determined

`Satisfaction` - evaluates how content the user was with the interaction:
- Very Satisfied: User expresses high satisfaction with the outcome
- Satisfied: User is generally content with the interaction
- Neutral: User shows neither satisfaction nor dissatisfaction
- Unsatisfied: User expresses some level of disappointment
- Very Unsatisfied: User shows strong disappointment or frustration
- Unknown: User satisfaction level cannot be determined

`Resolution` - tracks whether the user's query or issue was successfully addressed:
- Resolved Success: Query was fully addressed and resolved
- Resolved Partially: Some aspects of the query were resolved
- Not Resolved but Relevant: Agent provided relevant information but couldn't fully resolve
- Not Resolved and irelevant: Agent failed to provide relevant assistance
- Unknown: Resolution status cannot be determined

`Improvement` - assesses whether the agent's responses need enhancement:
- Need Improvement: Significant changes required in agent's responses
- Need some refinement: Minor adjustments could enhance performance
- No need for improvement: Agent performing optimally
- Unknown: Unable to determine if improvements are needed

## Using Analytics

The Analytics dashboard helps you monitor AI assistant conversations and identify areas for improvement. You can use various filters and metrics to analyze conversation performance and make necessary adjustments to your AI assistants.

### Key Features

- **Assistant Selection**: Filter analytics by specific assistant or view data across all assistants
- **Date Range Selection**: Analyze data within specific time periods
- **Interactive Charts**: Visualize performance metrics through various chart types
- **Detailed Tables**: View individual conversation data with expandable details
- **Performance Metrics**: Track scores, sentiment, satisfaction, and other key indicators
- **Export Options**: Download analytics data for external reporting

### Analytics Dashboard Components

1. **Assistant Selector**: Choose a specific assistant or view data for all assistants

2. **Date Range Picker**: Select the time period for your analysis

3. **Performance Overview**:
   - Average score per month chart showing performance trends
   - Performance statistics pie chart (good, average, poor)
   - Needs improvement statistics
   - Score distribution chart
   - Client intent statistics

4. **Detailed Metrics**:
   - Client satisfaction statistics
   - Suggestion statistics based on conversation analysis
   - Conversation table with detailed metrics for each session

### Common Use Cases

1. **Identifying Problem Areas**
   - Filter by assistants with lower performance scores
   - Review improvement suggestions for specific guidance
   - Access full conversations to understand context

2. **Performance Monitoring**
   - Track score trends over time with the monthly average chart
   - Identify patterns in user satisfaction and intent
   - Monitor resolution rates across different assistants

3. **Quality Improvement**
   - Use the needs improvement statistics to find areas requiring attention
   - Review conversations with low scores to understand common issues
   - Make targeted improvements to assistant configuration or training

### Example Workflow

1. Select an assistant from the dropdown menu (or choose "All assistants")
2. Set a relevant date range for analysis
3. Review the average score trend to identify performance patterns
4. Examine the performance and needs improvement pie charts
5. Check client intent and satisfaction statistics for insights
6. Review individual conversation data in the table
7. Make necessary adjustments to your assistant configuration
8. Monitor metrics over time to verify improvements

The Analytics module is a premium feature that requires specific subscription permissions. If you don't have access to Analytics, you'll see an upgrade prompt explaining how to enable this feature.
